Authentication

POST login

Autenticazione utente con username e password

🔐 Role: any 💾 State: utenti table
Request Body
  • username string - Username (es. m.bozzarelli)
  • password string - Password hash
Response
{ "token": "ABC123...", "user": { "id": "USR001", "nome": "Marcello", "ruolo": "admin" } }
POST logout

Logout e invalidazione token

🔐 Role: any 💾 State: token cache
Request
Header X-Token: token_value
Response
{ "ok": true, "message": "Logout successful" }

Urgenze (Emergencies)

POST createUrgenza

Crea nuova urgenza/emergenza con assegnazione

🔐 Role: admin 🔴 State: aperta
Fields
  • cliente_id string
  • macchina_id string
  • titolo string
  • descrizione string
  • tecnico_id string
  • priorita 1-5
POST assignUrgenza

Assegna urgenza a tecnico

🔐 Role: admin 🟡 State: assegnata
Fields
  • urgenza_id string
  • tecnico_id string
POST startUrgenza

Tecnico inizia intervento urgenza

🔐 Role: tecnico 🟠 State: in_corso
Fields
  • urgenza_id string
  • nota_inizio string (optional)
POST resolveUrgenza

Tecnico risolve urgenza con note finali

🔐 Role: tecnico 🟢 State: risolta
Fields
  • urgenza_id string
  • note_risoluzione string
  • ore_impiegate number
POST closeUrgenza

Admin chiude urgenza risolta

🔐 Role: admin ⚫ State: chiusa
Fields
  • urgenza_id string
  • nota_chiusura string (optional)
GET getUrgenze

Lista urgenze con filtri stato/tecnico/cliente

🔐 Role: any 📊 Response: array
Query Parameters
  • stato string - Filter by state
  • tecnico_id string - Filter by technician
  • limit number - Max 1000
