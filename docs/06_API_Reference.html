Getting Started

Syntoniqa v1.0 is a Field Service Management (FSM) platform for MRS Lely Center Emilia Romagna. The API runs on Cloudflare Workers with Supabase PostgreSQL backend.

https://syntoniqa-mrs-api.fieldforcemrser.workers.dev

Authentication

All endpoints (except login) require an X-Token header.

curl -H "X-Token: your_token_here" \
  https://syntoniqa-mrs-api.fieldforcemrser.workers.dev?action=getAll

Telegram Bot Commands

TG
/stato
Get current status overview
▼

Returns dashboard: open urgencies, today's interventions, SLA status, team availability

TG
/vado [n]
Accept an urgency task
▼

Shows 5 open urgencies. Reply /vado or /vado 2 to accept. Assigns and changes state to "assegnata".

TG
/incorso
Mark current urgency as in progress
▼

Changes assigned urgency state from "assegnata" to "in_corso".

TG
/risolto [note]
Resolve and close urgency
▼

Closes current urgency. Changes state to "risolta" then "chiusa". Optional notes saved as resolution.

TG
/ordine [code] [qty] [client]
Create spare parts order
▼

Creates order: /ordine PAR-001 2 CLI_123. Admin receives notification.

TG
Photo/Text Message
Send photo or text for AI analysis
▼

Send photo or text description. Gemini AI analyzes and automatically creates urgency if confirmed.

State Machine

Urgenze States

aperta → assegnata → schedulata → in_corso → risolta → chiusa

Reject: assegnata/schedulata → aperta

Piano States

pianificato → in_corso → completato
OR
pianificato → annullato (then annullato → pianificato)
